List of changes

  • In our initial design from P2 our pages varried in width, this has been changed to make the pages look more uniform and aesthetically pleasing. We achieved this by adding a main element with a set width.
  • We changed some filenames from P2:
    • order.html is now called order-now.html
  • The boxes on the contact-us page no longer have a set width and height in pixles. The width of the box is determined by the main element width and the height is determined by the svg height and padding.
  • Added a dessert and drinks section to our menu page.
  • We didn't use JavaScript to implement the map functionality, hence there is no map.js file.
  • Our layout.js turned into footer.js and header.js
  • Our button.js is integrated into our header.js
  • Our widget.js is now called info-pill.js
  • Small aesthetic adjustments to our menu page and our about-us page to make it more aeshtetically pleasing.
  • We decided not to use a JSON-file for our menu-page, instead we stored all the menu items in an array within the menu.js and itterated through the array with a map function to update the menu-page html.

Responsive web design

Our website was designed with desktops and laptops in mind. The website is responsive when the window scales, however it does not have a specific layout for mobile phones. This is a feature we would prioritise in the future.

Modularity

To achieve the modularity requirement we made our footer and header in JavaScript so you can import the code for every page. All our CSS is stored in our global.css file. We worked together to achieve the modularity requirement, since Brage M and Philip had the most experience in web-design they helped the rest of the group along the way.
